# 苯乙烯装置监测系统 —— 实验方案与测试指南

> 版本：v1.0 · 日期：2026-05-15  
> 本文档同时作为**开发方案**与**测试操作手册**使用。

---

## 一、工艺背景与系统目标

苯乙烯（Styrene, ST）工业生产的主流路线是**乙苯催化脱氢**：

```
C₆H₅-CH₂CH₃  ──(催化剂, 550-620°C)──►  C₆H₅-CH=CH₂ + H₂
    乙苯 (EB)                                  苯乙烯 (ST)
```

关键工艺条件：
- 反应温度：550–620°C（绝热降温，进口高出口低）  
- 操作压力：60–100 kPa（接近常压，微负压抑制副反应）  
- 蒸汽稀释：水蒸气/乙苯摩尔比 ≈ 2.5–3.0（降低乙苯分压，抑制结焦）

**本实验目标**：

1. 在主模型中建立传感器类型、传感器、设备类型、设备数据
2. 配置虚拟设备模拟器，仿真苯乙烯装置的实时工艺数据
3. 开发 `st_plant` 插件，实现数据绑定、快照、SSE 实时推送
4. 在 P&ID 画板中可视化展示工艺流程
5. 端到端测试验证完整数据链路

### 3.1 传感器类型（SensorType）

共需 2 种类型（若已有同名类型可复用）。

#### 类型 A：工业温压传感器

| 字段 | 值 |
|------|----|
| SensorType_id | `ITTP-V1` |
| name | `工业温压传感器` |
| description | 测量管路/容器温度（°C）与压力（kPa），适用于工业工艺监测 |
| data_fields | `["temperature", "pressure"]` |
| config_parameters | `["samplingInterval", "statusReportInterval", "is_enabled"]` |
| commands | 见下方 JSON |

commands JSON：

> **格式约定**：
> - `mqtt_message` 必须是 JSON **对象**，会被原样推到 MQTT 控制主题；设备端通过 `payload.command` 区分动作，所以每条都要带 `"command": "<动作>"`
> - `params` 必须是**字符串数组**，每个字符串是占位符名，对应 `mqtt_message` 里 `"{xxx}"` 形式的变量
> - 如需参数 schema（type/min/max）给前端表单用，请另放到 `params_schema` 字段，不要污染 `params`

```json
{
  "enable":  {
    "mqtt_message": {"command": "enable"},
    "description": "启用传感器",
    "params": []
  },
  "disable": {
    "mqtt_message": {"command": "disable"},
    "description": "禁用传感器",
    "params": []
  },
  "set_interval": {
    "mqtt_message": {
      "command": "set_data_interval",
      "interval": "{interval}"
    },
    "description": "设置采样间隔（秒，5-3600）",
    "params": ["interval"]
  }
}
```

**API 操作**（使用 DRF Browsable API 或 curl）：
```bash
curl -X POST http://localhost:8000/api/sensors/sensor-types/ \
  -H "Authorization: Token <your_token>" \
  -H "Content-Type: application/json" \
  -d '{
    "SensorType_id": "ITTP-V1",
    "name": "工业温压传感器",
    "description": "测量管路/容器温度（°C）与压力（kPa），适用于工业工艺监测",
    "data_fields": ["temperature", "pressure"],
    "config_parameters": ["samplingInterval", "statusReportInterval", "is_enabled"],
    "commands": {
      "enable":  {"mqtt_message": {"command": "enable"},  "description": "启用传感器", "params": []},
      "disable": {"mqtt_message": {"command": "disable"}, "description": "禁用传感器", "params": []},
      "set_interval": {
        "mqtt_message": {"command": "set_data_interval", "interval": "{interval}"},
        "description": "设置采样间隔（秒，5-3600）",

  }'
```

## 四、第二步：虚拟设备模拟器配置

### 4.1 创建 Manifest 文件

新建文件：`simulation/manifests/st_plant.yaml`

```yaml
# 苯乙烯装置 (ST Plant) 虚拟节点清单
# 运行方式：python run.py -m st_plant
# 可与 default 合并：python run.py -m default -m st_plant

name: st_plant
description: 苯乙烯装置模拟节点（6 路传感器 + 2 台泵）

nodes:

  # ── 传感器 ────────────────────────────────────────────────

  # 1. 反应器进口温压（高温高压侧，进口温度约 560-620°C）
  - module: temp_pressure_sensor
    id: ST-TP-R01-IN
    sampling_interval: 15
    status_report_interval: 60
    waveforms:
      temperature:
        type: sine
        min: 560.0
        max: 620.0
        period: 3600
        jitter: 2.0
      pressure:
        type: random_walk
        start: 88.0
        step: 0.8
        bounds: [75.0, 100.0]

  # 2. 反应器出口温压（脱氢吸热，温度比进口低 50-80°C）
  - module: temp_pressure_sensor
    id: ST-TP-R01-OUT
    sampling_interval: 15
    status_report_interval: 60
    waveforms:
      temperature:
        type: sine
        min: 490.0
        max: 555.0
        period: 3600
        phase: 600        # 与进口错相，体现传输时延
        jitter: 2.0
      pressure:
        type: random_walk
        start: 65.0
        step: 0.6
        bounds: [55.0, 78.0]

  # 3. 换热器出口温压（废热回收后，进入冷凝器前，约 80-120°C）
  - module: temp_pressure_sensor
    id: ST-TP-HE01
    sampling_interval: 20
    status_report_interval: 60
    waveforms:
      temperature:
        type: sine
        min: 85.0
        max: 115.0
        period: 1800
        jitter: 1.0
      pressure:
        type: random_walk
        start: 60.0
        step: 0.4
        bounds: [50.0, 72.0]

  # 4. 冷凝器出口温压（产品冷凝，约 25-45°C）
  - module: temp_pressure_sensor
    id: ST-TP-CD01
    sampling_interval: 20
    status_report_interval: 60
    waveforms:
      temperature:
        type: random_walk
        start: 35.0
        step: 0.5
        bounds: [25.0, 45.0]
      pressure:
        type: sine
        min: 50.0
        max: 62.0
        period: 900
        jitter: 0.3

  # 5. 乙苯进料流量（工艺设计值约 50 L/min，允许 ±10% 波动）
  - module: flow_sensor
    id: ST-FLOW-FEED
    sampling_interval: 10
    status_report_interval: 60
    waveforms:
      flow_rate:
        type: sine
        min: 45.0
        max: 55.0
        period: 1200
        jitter: 0.8

  # 6. 稀释蒸汽流量（蒸汽/乙苯摩尔比 ≈ 2.7，约 130-140 L/min）
  - module: flow_sensor
    id: ST-FLOW-STEAM
    sampling_interval: 10
    status_report_interval: 60
    waveforms:
      flow_rate:
        type: sine
        min: 128.0
        max: 142.0
        period: 1200
        phase: 300          # 与进料略错相
        jitter: 1.5

  # ── 设备 ─────────────────────────────────────────────────

  # 7. 乙苯进料泵（额定 22 kW）
  - module: pump
    id: ST-PUMP-FEED
    status_report_interval: 30
    max_power_kw: 22.0

  # 8. 冷凝水循环泵（额定 15 kW）
  - module: pump
    id: ST-PUMP-CW
    status_report_interval: 30
    max_power_kw: 15.0
```

### 4.2 波形设计说明

| 传感器 | 字段 | 波形类型 | 含义 |
|--------|------|----------|------|
| 反应器进口 | temperature | sine | 周期 1h 的工况波动（负荷变化） |
| 反应器进口 | pressure | random_walk | 背压随机扰动 |
| 反应器出口 | temperature | sine + phase | 比进口延迟 600s，模拟停留时间 |
| 换热器出口 | temperature | sine | 较快周期（30min）反映热媒流量变化 |
| 冷凝器出口 | temperature | random_walk | 慢变量，受环境温度影响 |
| 进料流量 | flow_rate | sine | DCS 给定值附近波动 |
| 蒸汽流量 | flow_rate | sine + phase | 与进料协调，但有滞后 |

### 5.5 signals.py

```python
import logging
from django.db.models.signals import post_save
from django.dispatch import receiver

log = logging.getLogger(__name__)

PLUGIN_CODE = "st_plant"


def _handle_sensor_data(sender, instance, created, **kwargs):
    if not created:
        return
    from .models import STPSensorBinding
    from services.realtime.latest_values import ingest_sensor_data

    try:
        binding = STPSensorBinding.objects.get(
            sensor_id=instance.sensor_id,
            is_visible=True,
        )
    except STPSensorBinding.DoesNotExist:
        return

    ingest_sensor_data(PLUGIN_CODE, instance, binding)
    log.debug(f"[st_plant] ingest {instance.sensor_id} tag={binding.tag}")


def _connect():
    try:
        from sensors.models import SensorData
        post_save.connect(_handle_sensor_data, sender=SensorData, weak=False)
    except Exception as e:
        log.warning(f"[st_plant] signals connect failed: {e}")


_connect()
```

```python
@api_view(["GET"])
@permission_classes([AllowAny])   # SSE 无法附带 Authorization header
def stream(request):
    """
    SSE 实时数据流。
    先发一次 snapshot 数据（event: snapshot），
    再持续消费 Broadcaster 的广播队列（event: update）。
    """
    def _generator():
        q = queue.Queue(maxsize=50)
        Broadcaster.subscribe(PLUGIN_CODE, q)
        try:
            # 先发快照
            snap = LatestValuesCache.get(PLUGIN_CODE) or {}
            yield f"event: snapshot\ndata: {json.dumps(snap)}\n\n"
            # 持续推送
            while True:
                try:
                    msg = q.get(timeout=25)
                    yield f"event: update\ndata: {json.dumps(msg)}\n\n"
                except queue.Empty:
                    yield ": heartbeat\n\n"
        finally:
            Broadcaster.unsubscribe(PLUGIN_CODE, q)

    resp = StreamingHttpResponse(_generator(), content_type="text/event-stream")
    resp["Cache-Control"] = "no-cache"
    resp["X-Accel-Buffering"] = "no"
    return resp
```

### 8.4 异常情况处理

| 问题 | 排查方式 |
|------|----------|
| 模拟器连接 MQTT 失败 | 检查 `simulation/config.yaml` 的 host/port；确认 mosquitto 运行 |
| SensorData 无新记录 | 检查 Django MQTT 服务是否订阅了相关主题；查看后端日志 |
| snapshot 为空 | 确认绑定已创建（`/api/plugins/st_plant/sensor_bindings`）；等待第一次 ingest |
| SSE 断开 | Nginx 配置需关闭 proxy_buffering；开发环境直连 8000 端口无此问题 |
| 画板节点不更新 | 检查前端 useSSE.js 的 status ref 是否为 open；查看浏览器 Network → EventStream |
