# 更新日志

---

## v0.2 (2026-03)

### 平台配置 (platform_settings)

- **新增 platform_settings 应用**：Key-Value 存储非敏感、可调配置（MQTT、设备离线等）
- **配置读取优先级**：数据库（platform_settings）> 环境变量 > 默认值
- **get_config**：`config/platform_config.py` 统一读取，支持 str/int/bool/float 类型
- **seed_platform_config**：管理命令将 .env 默认值写入数据库，仅创建不存在的 key，支持 `--force` 覆盖
- **前端系统设置**：超级用户可在「平台配置」区块增删改配置
- **settings.py**：MQTT、设备离线等配置改为通过 `get_config` 读取，使用 `SimpleLazyObject` 延迟加载

### 权限体系

- **超级用户 (is_superuser)**：平台配置、自动化规则的新增/编辑/删除
- **工作人员 (is_staff)**：传感器/设备及类型的增删改、传感器/设备命令、自动化执行/启动/停止
- **非工作人员**：仅查看，无任何增删改操作
- **注册**：只能注册非工作人员（`is_staff=False, is_superuser=False`）
- **IsSuperuser**：新增 `config/permissions.py` 权限类

### 前端权限控制

- 平台配置：`isSuperuser` 控制新增/编辑/删除
- 自动化：`isSuperuser` 控制新建规则、编辑、删除；`isStaff` 控制执行/启动/停止按钮显示
- 传感器/设备：`isStaff` 控制类型管理、添加按钮、命令控制区
- 传感器/设备详情：命令控制卡片仅工作人员可见
- 自动化详情：执行控制台（单次执行、启动轮询、停止轮询）仅工作人员可见

### 功能迁移

- 传感器类型管理：从系统设置页迁移到传感器管理页
- 设备类型管理：从系统设置页迁移到设备管理页
- 添加设备/传感器按钮：空状态时放在空卡片内，有数据时放在筛选栏

### Docker 部署

- 启动命令：`migrate` 后增加 `seed_platform_config`，自动初始化平台配置
- 文档：更新 `docs/deployment/docker.md`，修正 MySQL 架构、项目路径、MQTT 配置说明
