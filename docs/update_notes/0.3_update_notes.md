# 更新日志

---

## v0.3 (2026-03)

### Docker 端口配置

- **前端**：`localhost:8080`（`FRONTEND_PORT`）
- **Admin 管理界面**：`localhost:8081`（`ADMIN_PORT`），直连后端
- **backend 端口映射**：新增 `${ADMIN_PORT:-8081}:8000`

### Admin 静态文件

- **Whitenoise**：新增依赖，Gunicorn 下正确提供 Admin CSS/JS
- **STATIC_ROOT**：`staticfiles`，启动时执行 `collectstatic --noinput`

### 平台配置重构

- **default_config.json**：`platform_settings/default_config.json` 定义默认配置，与 .env 合并后写入数据库
- **.env 精简**：仅保留启动必备项（DB、MQTT 连接、SECRET_KEY 等），其他配置在 JSON 中
- **get_config**：仅从 platform_settings 数据库读取，不再回退环境变量或 JSON
- **seed_platform_config**：从 JSON 读取配置项，env_key 非空时可由 .env 覆盖

### 数据留存与清理

- **配置项**：`sensor_data_retention_days`、`device_data_retention_days`（默认 30 天）
- **cleanup_old_data**：管理命令按配置清理过期传感器/设备数据，支持 `--dry-run`
- **启动时自动执行**：seed_platform_config、cleanup_old_data 在服务启动时各执行一次

### Reload 机制（无需重启）

- **POST /api/platform-configs/reload/**：使配置修改生效，MQTT 自动重连
- **mqtt_service**：连接参数从 `get_config` 读取，支持 reload 后重连
- **前端**：「使配置生效」按钮

### 前端可用命令

- **可用命令卡片**：系统设置页新增卡片，与 MQTT 连接状态同风格
- **清理过期数据**：按钮触发 `cleanup_old_data`，仅超级用户可见
- **API**：`POST /api/platform-configs/cleanup-old-data/`

### 日志系统

- **轮转策略**：`RotatingFileHandler` 改为 `TimedRotatingFileHandler`，按天轮转，默认保留 2 天
- **platform_settings.log**：新增平台配置专用日志（seed、cleanup、reload）
- **platform_settings logger**：seed_platform_config、cleanup_old_data、views 使用统一 logger 记录

### 文档

- **更新**：`platform_settings_guide.md`、`platform_settings_design.md`、`logsystem_design.md`、`logsystem_guide.md`
- **更新**：`docker.md`、`README.md` 中 Admin 端口与配置说明
