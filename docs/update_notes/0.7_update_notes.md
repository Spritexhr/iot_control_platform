# 0.7 版本更新日志

## 变更概览

本次更新围绕**部署 / 配置体验**与**前端可维护性**两条主线推进：

- **配置系统统一**：废弃 `default_config.json` 与 `seed_platform_config` 命令，改由 `defaults.py` + `configure` 管理命令承载所有运行期配置；MQTT 等业务参数全部从 `.env` 迁入数据库，`.env` 仅保留进程级启动必备项
- **首次部署引导**：`configure` 提供 wizard / `--init` / `--set` / `--unset` / `--list` 子模式；启动期自动 `--init` 补缺失项；前端检测默认 broker 时弹首次部署提示
- **系统设置页重写**：通用 CRUD 表 → 按分类分组的表单视图（数字/字符串/密码差异化控件、MQTT 测试连接、保存反馈闭环），自定义 key 折叠到"高级配置"
- **Token 主动续期**：axios 请求拦截器解析 JWT exp，过期前 30s 主动 refresh，消除 access token 过期窗口里 Django 端持续打 `Unauthorized` WARNING 的问题
- **i18n 体系升级**：`locale store` 支持参数插值，新增模块级 `staticT` 让 axios 拦截器等非组件代码也能 i18n
- **端口语义修正**：宿主机暴露端口（FRONTEND_PORT / ADMIN_PORT）从默认 8080/8081 改为 48080/48081，避开常见冲突；MQTT 仍走 1883 EMQX 标准端口

## 一、配置系统重构

### 背景

0.6 之前同一份配置散在三处：`.env`（环境变量）、`platform_settings/default_config.json`（种子 JSON）、`PlatformConfig` 表（运行期值）。导致：

- MQTT 密码进了 `.env`，提交风险
- `seed_platform_config` 命令只懂 JSON，wizard 体验缺失
- 用户不清楚改哪份才会真正生效，且公网地址容易误写进 `.env.example`

### 变更

#### 后端

- **新增** `platform_settings/defaults.py`：所有可调配置以 Python 列表形式声明，每条带 `key / default / category / description / secret / value_type` 元数据，作为种子值与 wizard 的唯一信源
- **新增** 管理命令 `configure`，支持四种模式：

  | 模式 | 用途 |
  |---|---|
  | `configure`（无参） | 交互式 wizard，按分类引导填值，密码使用 `getpass` 不回显 |
  | `configure --init` | 仅写入缺失的 key，已存在的不动；启动期默认调用 |
  | `configure --set k=v [--set ...]` | 单键设置，CI / 脚本化 |
  | `configure --unset key` | 删除某个 key |
  | `configure --list` | 当前 DB 值（密码自动打码） |

- **`configure` 默认在写入后调用** `mqtt_service.reconnect()` 让新配置即时生效；启动期用 `--no-reload` 跳过这一步避免重复连接
- **删除** `platform_settings/default_config.json`、`management/commands/seed_platform_config.py`、`配置说明示例/示例.txt`
- **修改** `sensors/apps.py`：启动钩子从 `seed_platform_config` 改为 `configure --init --no-reload`；同时**从启动钩子移除** `cleanup_old_data`（应通过 cron / 后台任务调度，不应每次进程启动都扫表）

#### 前端

- **新增** `GET /platform-configs/schema/`（仅认证）：把 `defaults.py` 暴露给前端，返回 `{items[], known_keys[]}`，每个 item 含 `key/category/default/description/secret/type`，用于按分类渲染表单
- **新增** `POST /platform-configs/test-mqtt/`（仅 superuser）：用临时 paho client 探测连接，5 秒超时，返回 `{success, message}`，不影响在跑的 `mqtt_service`

#### `.env` 极简化

| 变量 | 0.6 | 0.7 |
|---|---|---|
| `MQTT_BROKER` / `MQTT_PORT` / `MQTT_USERNAME` / `MQTT_PASSWORD` | 在 `.env` | **移到 DB**，`configure` 维护 |
| `DEVICE_OFFLINE_TIMEOUT` 等 | `.env` | **移到 DB** |
| `SECRET_KEY` / `DEBUG` / `ALLOWED_HOSTS` | `.env` | 保留（进程级） |
| `DB_*` | `.env` | 保留（鸡生蛋问题） |
| `FRONTEND_PORT` / `ADMIN_PORT` | 8080/8081 | **48080/48081** |

### 设计要点

- **为什么不一并消除 `.env`**：DB 连接信息有"鸡生蛋"问题（连 DB 之前需要先知道连哪个 DB）；密钥放 DB 也增加日志泄露风险
- **`secret: True` 字段 wizard 不回显**：使用 `getpass.getpass` + `--list` 中以 `***` 打码
- **`_coerce(raw, default)`**：wizard 输入的字符串按 `default` 的 Python 类型自动转换（int / float / bool / str / list / dict）

### 文件变更

| 文件 | 变更 |
|---|---|
| `platform_settings/defaults.py` | 新增 |
| `platform_settings/management/commands/configure.py` | 新增 |
| `platform_settings/management/commands/seed_platform_config.py` | **删除** |
| `platform_settings/default_config.json` | **删除** |
| `platform_settings/views.py` | 新增 `schema` / `test_mqtt` actions |
| `sensors/apps.py` | 启动钩子改 `configure --init --no-reload`；移除 `cleanup_old_data` |
| `config/settings.py` | CORS / CSRF 默认源加 48080/48081 |
| `.env.example` | 重写为 9 项（`MQTT_*` 全部移除） |
| `docker-compose.yml` | 移除 `MQTT_*` env，端口默认 48080/48081，启动命令插入 `configure --init --no-reload` |
| `docker-compose.test.yml` | 同上 |

## 二、系统设置页重写

### 背景

0.6 设置页是一张**通用 CRUD 表**：所有 key 一行，value 用 JSON textarea 编辑，可任意"新增 / 删除"包括 `mqtt_broker` 这种系统 key。问题：

1. 预定义 key 被当成自由 CRUD，用户能误删核心配置
2. 字段语义丢失：密码不打码、端口没数字校验、布尔无开关
3. wizard 的引导体验在 UI 上未复刻
4. 改完不知道是否真的生效（无反馈闭环）
5. 首次部署没引导提示

### 变更

#### `SettingsView.vue` 整页重写

- **首次部署 banner**：检测 `mqtt_broker == "127.0.0.1"` 时顶部显示警告条
- **MQTT 状态卡**：保留原有"连接中 / 已连接"展示
- **分组配置表单**：MQTT / 设备 / 数据保留各自一张卡片，按 schema 的 `type` 与 `secret` 自动选控件：
  - `integer` / `number` → `el-input-number`，`*_port` 自动限制 1–65535
  - `boolean` → `el-switch`
  - `secret: true` → 默认显示"已设置"占位符 + "修改"按钮，点击才进入 password 输入
  - `string` → 普通 `el-input`
- **测试连接**按钮（仅 MQTT 卡）：用当前表单里（含未保存的）broker/port/密码探测一次连接，调 `/platform-configs/test-mqtt/`
- **保存本组**按钮：脏检查（`fieldDirty(field)`），不脏就 disable；保存成功后刷新本地 `_original` 以重置脏态
- **可用命令卡**：`cleanup_old_data` 仍可手动触发；"使配置生效"（reload MQTT）从配置编辑区移到此处
- **高级 / 自定义配置**：折叠区，默认关闭。沿用旧 CRUD 表 + JSON 编辑器，但只显示**非预定义** key（`!knownKeys.has(key)`）；不允许把自定义 key 命名成已知 key（前端拦截）

#### 前端 API（`api/platformConfig.js`）

- 新增 `getConfigSchema()` → `GET /platform-configs/schema/`
- 新增 `testMqttConnection(payload)` → `POST /platform-configs/test-mqtt/`

### 设计要点

- **"分组表单 + 折叠自定义"双层结构**：80% 用户只需要改 wizard 那几个 key，留高级折叠让运维 / 插件开发者继续能加自定义 key（响应你的"为之后配置新功能"诉求）
- **保存粒度按分组**：不强制全表单一起保存，避免 MQTT 测试还没通过就被迫保存
- **首次部署 banner 用启发式而非新字段**：避免引入 `platform_initialized` 标志位带来的迁移成本；只要用户把 broker 改成实际地址，banner 就消失

### 文件变更

| 文件 | 变更 |
|---|---|
| `platform_settings/views.py` | 新增 `schema` / `test_mqtt` actions |
| `frontend/src/api/platformConfig.js` | 新增 `getConfigSchema` / `testMqttConnection` |
| `frontend/src/views/settings/SettingsView.vue` | 整页重写 |

## 三、JWT 主动续期

### 背景

启动后 Django 日志频繁出现：

```
WARNING ... [django.request] Unauthorized: /api/auth/profile/
WARNING ... [django.request] Unauthorized: /api/mqtt/status/
```

根因：access token 过期（SimpleJWT 默认 2h）后，前端仍带旧 token 发请求 → Django 返回 401 → 响应拦截器走 refresh + retry → 用户无感，但 Django 端**先打了一条 WARNING** 才轮到拦截器修复。

### 变更

`api/index.js` 加**请求拦截器层主动续期**：

- `decodeJwtExp(token)`：解析 JWT 第二段拿 `exp` 秒
- `accessTokenAboutToExpire(token)`：剩余 ≤ 30 秒视为即将过期
- 请求发出前，若即将过期就先 `await refreshAccessToken()` 拿新 token 再发
- **并发请求共享同一个 refresh promise**（`proactiveRefreshing`），不会重复打 `/auth/refresh/`
- 跳过 `/auth/login` 与 `/auth/refresh` 自身，避免循环

原响应拦截器的 401 → refresh → retry 流程**保留**作为兜底（防止 token 在 30 秒窗口里恰好过期）。

### 文件变更

| 文件 | 变更 |
|---|---|
| `frontend/src/api/index.js` | 新增 `decodeJwtExp` / `accessTokenAboutToExpire` / `refreshAccessToken`；请求拦截器加主动续期 |

## 四、i18n 体系升级（A+B 阶段）

### 背景

0.6 的 `stores/locale.js` 是个简化版 i18n：`t(path)` 闭包读 `locale.value`，仅在 setup 中可用。问题：

1. **不支持参数插值**：动态消息只能拼字面量（`${count} 是预定义 key`）
2. **非组件环境拿不到**：axios 拦截器、router guard 直接写中文（`'登录已过期，请重新登录'` / `'请求失败'`）
3. 缺 key 时静默返回 path，调试不易发现漏译

> 全项目目前仍有 ~345 处硬编码中文跨 19 个 Vue 文件（传感器卡片 / 设备详情 / 自动化 / 数据可视化等），**不在本次范围**，留作 0.8 单独迁移。

### 变更（locale store 升级）

`stores/locale.js` 把翻译逻辑提到模块顶层：

```js
function resolvePath(messages, locale, path) { ... }  // 树状路径解析
function interpolate(template, params) { ... }        // {key} 模板插值

export function staticT(path, params, localeOverride) {
  // 读 localStorage 取 locale，找不到 key 时回退中文，最后做插值
}

export const useLocaleStore = defineStore('locale', () => {
  const locale = ref(...)
  function t(path, params) { return staticT(path, params, locale.value) }
  ...
})
```

带来的能力：

- **参数插值**：`ls.t('settings.predefinedKeyHint', { key: 'mqtt_broker' })` → `mqtt_broker 是预定义配置项...`
- **非组件可用**：`import { staticT } from '@/stores/locale'` 在 `api/index.js` 等处直接用
- **缺 key 兜底**：英文环境漏 key 自动回退中文，不再显示 path 字符串
- **响应式 vs 静态**：组件继续用 `ls.t()`（locale 切换响应式刷新）；非组件环境用 `staticT()`（每次读 localStorage，立即生效）

### 新增 i18n key

中英各 4 条：

| key | 用途 |
|---|---|
| `settings.cannotDeletePredefined` | 阻止删除预定义 key 的提示 |
| `settings.predefinedKeyHint` | 阻止把自定义 key 命名为已知 key（带 `{key}` 插值） |
| `api.sessionExpired` | axios 401 后跳登录页的弹窗 |
| `api.requestFailed` | axios 通用错误兜底 |

### 替换硬编码

- `api/index.js`：`'请求失败'` / `'登录已过期...'` → `staticT('api.requestFailed')` / `staticT('api.sessionExpired')`
- `SettingsView.vue`：两处中文字面量 → `ls.t(...)`

### 文件变更

| 文件 | 变更 |
|---|---|
| `frontend/src/stores/locale.js` | 抽出 `staticT`、加插值、加兜底；新增 8 条 i18n key |
| `frontend/src/api/index.js` | 接入 `staticT` |
| `frontend/src/views/settings/SettingsView.vue` | 两处硬编码替换为 `ls.t()` |

### 后续如何用

**组件里**（响应式，切语言自动重渲染）：

```js
const ls = useLocaleStore()
ls.t('common.save')
ls.t('msg.foundCount', { n: 3 })
```

**非组件文件里**（axios、router guard、工具函数）：

```js
import { staticT } from '@/stores/locale'
ElMessage.error(staticT('api.requestFailed'))
```

**加新 key**：`stores/locale.js` 的 `messages.zh.<分组>` 与 `messages.en.<分组>` 同时加 → 在调用处用 `ls.t('分组.key')` 或 `staticT('分组.key')`。

## 五、升级步骤

### Docker 部署

```bash
git pull
docker compose down
docker compose up -d --build      # backend 镜像需重 build：configure 命令、defaults.py、views.py 都变了
                                    # frontend 镜像需重 build：locale store / settings 页 / axios 都变了

# 首次部署或想改 MQTT 等配置：
docker compose exec backend python manage.py configure       # 交互式 wizard
# 或：
docker compose exec backend python manage.py configure --set mqtt_broker=192.168.1.10 --set mqtt_port=1883
```

### 测试环境（源码挂载）

```bash
docker compose -f docker-compose.test.yml down
docker compose -f docker-compose.test.yml up -d
# 后端容器启动会跑 configure --init --no-reload，把 defaults.py 写入 DB
```

### 本地（非 Docker）

```bash
git pull
cd iot_control_platform
python manage.py migrate              # 0.7 没新增迁移
python manage.py configure --init     # 写入默认值（仅补缺失）
python manage.py configure            # 可选：交互式 wizard 改 MQTT 等

# 重启 runserver / gunicorn

cd ../frontend
npm install
npm run dev   # 或 npm run build
```

### 注意事项

- **`.env` 不再承载 MQTT**：从 0.6 升级到 0.7 时，原 `.env` 里的 `MQTT_BROKER` / `MQTT_PORT` 等会被忽略；启动后用 `configure` 把这些值写入 DB
- **端口默认值变化**：原来用 8080 / 8081 直接通的环境，0.7 默认改为 48080 / 48081；如要保留旧端口，在 `.env` 显式设置 `FRONTEND_PORT=8080` / `ADMIN_PORT=8081`
- **`cleanup_old_data` 不再随启动自动跑**：如需定期清理，建议加 cron 或 systemd timer，调用 `python manage.py cleanup_old_data` / 或前端"系统设置 → 可用命令 → 清理过期数据"按钮手动触发

## 六、设计要点速查

1. **配置三段统一为两段**：`defaults.py`（种子 + meta）→ `PlatformConfig` DB（运行值）；`.env` 只剩进程级启动必备项
2. **wizard 是唯一的"友好首次配置入口"**：CLI 的 `configure` 与前端 SettingsView 表单两端等价
3. **MQTT 测试连接独立 paho 客户端**：不影响在跑的 `mqtt_service`，5 秒超时
4. **JWT 主动续期 + 401 兜底双保险**：常态走主动 refresh（无 WARN），边界 case 走响应拦截器
5. **i18n 提供两种调用形态**：`ls.t()`（响应式，组件用）与 `staticT()`（任何地方都能 import）
6. **首次部署 banner 用启发式**：`mqtt_broker == 127.0.0.1` 即提示，避免引入新字段与迁移

## 七、已知限制 / 后续可优化

- **全项目 i18n 普及（约 345 处硬编码跨 19 个 Vue 文件）**：本次仅完成基础设施 + 设置页 + 全局错误消息；传感器 / 设备 / 自动化 / 数据可视化等界面留作 0.8 单独迁移
- **MQTT 密码 DB 明文存储**：当前是 `JSONField` 直接存；后续可考虑用 Django `cryptography` 包做应用层加密，或交由系统密钥管理服务
- **首次部署 banner 不能区分"用户故意把 broker 设回本地"**：启发式判断会在合法场景下也显示。如果体验上有干扰，可改为 `platform_initialized` 标志位（一次性数据迁移）
- **`configure --set` 不支持 list / dict**：CLI 单键模式只支持 scalar（string / int / float / bool）；list / dict 类型仍需走 wizard 或 admin
- **JWT 主动续期阈值固定 30 秒**：未参数化，若网络延迟较大可调大；目前以 SimpleJWT 默认 2h access token 为前提
