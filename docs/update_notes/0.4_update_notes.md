# 0.4 版本更新日志

## 变更概览

本次更新主要包含：命令下发服务重构（提取公共基类）、MQTT 自动重连、生产环境安全加固、健康检查端点、数据清理分批删除、查询性能优化。

---

## 一、BaseCommandSendService 基类重构

### 背景

传感器和设备的命令下发服务（`SensorCommandSendService`、`DeviceCommandSendService`）存在大量重复代码：命令构造、占位符替换、check_code 注入/校验、等待确认逻辑完全一致，仅模型类和字段名不同。

### 变更

- **新增** `services/base_command_send_service.py`：提取公共基类 `BaseCommandSendService`
  - 封装 `send_command`、`send_custom_command`、`send_custom_command_with_make_sure`
  - 封装 `_apply_params_to_message`、`_inject_check_code`、`_verify_check_code`
  - 封装 check_code 缓存（`_PENDING_CHECK_CODES`）和等待事件（`_WAITING_EVENTS`）
  - 子类仅需定义 `model_class`、`id_field_name`、`type_field_name` 三个类属性
- **重构** `sensor_command_send_service.py`：从 226 行精简至 25 行，继承 `BaseCommandSendService`
- **重构** `device_command_send_service.py`：同样精简，继承 `BaseCommandSendService`
- **接口统一**：传感器和设备的 `send_custom_command_with_make_sure` 均统一使用 `time_out` 参数

### 文件变更

| 文件 | 变更 |
|-----|------|
| `services/base_command_send_service.py` | 新增 |
| `services/sensors_service/sensor_command_send_service.py` | 重构为继承基类 |
| `services/devices_service/device_command_send_service.py` | 重构为继承基类 |

## 二、MQTT 自动重连（指数退避）

### 背景

MQTT 服务断线后无法自动恢复，需要人工介入重启。

### 变更

- 启用 paho-mqtt 内置 `reconnect_delay_set()`，断线后自动重连
- 指数退避策略：延迟从 1 秒增长至上限 120 秒（1→2→4→8→...→120）
- 连接成功后自动重置重连延迟为最小值
- `_on_disconnect` 回调仅负责日志记录和状态更新，不再手动触发重连
- `stop()` 会完全停止客户端和自动重连

### 文件变更

| 文件 | 变更 |
|-----|------|
| `services/mqtt_service.py` | 新增自动重连配置与回调逻辑 |

## 三、生产环境安全配置

### SECRET_KEY 管理

- 生产环境（`DEBUG=False`）未设置 `SECRET_KEY` 环境变量时，抛出 `ImproperlyConfigured` 阻止启动
- 开发环境允许使用固定不安全默认值

### 安全头

`DEBUG=False` 时自动启用：

- `SECURE_BROWSER_XSS_FILTER = True`
- `SECURE_CONTENT_TYPE_NOSNIFF = True`
- `X_FRAME_OPTIONS = DENY`
- `SESSION_COOKIE_SECURE` / `CSRF_COOKIE_SECURE` 可通过环境变量控制

### CORS 扩展

支持通过 `EXTRA_CORS_ORIGINS` 环境变量追加额外 CORS 源（逗号分隔）。

### API 节流

新增全局节流配置：

| 类别 | 速率 |
|-----|------|
| 匿名用户 | 30 次/小时 |
| 认证用户 | 100 次/小时 |

### 文件变更

| 文件 | 变更 |
|-----|------|
| `config/settings.py` | SECRET_KEY 安全策略、安全头、CORS 扩展、节流配置 |

## 四、健康检查端点

### 新增 `GET /health/`

无需认证，供监控系统和负载均衡器使用。

返回服务状态、数据库连接状态、MQTT 连接状态：

```json
{
  "status": "ok",
  "checks": {
    "database": "ok",
    "mqtt": "connected"
  },
  "timestamp": "2026-04-12T10:30:00+08:00"
}
```

- 正常时 HTTP 200，异常时 HTTP 503

| 文件 | 变更 |
|-----|------|
| `config/api_views.py` | 新增 `health_check` 视图 |
| `config/urls.py` | 新增 `/health/` 路由 |

## 五、数据清理分批删除

## 六、查询性能优化（N+1 问题）

### 变更

- **DeviceType / SensorType**：查询集使用 `annotate(_device_count=Count('devices'))` / `annotate(_sensor_count=Count('sensors'))`，避免每个类型的计数触发额外查询
- **Device / Sensor 列表**：使用 `prefetch_related('data_records')` 预取最新数据
- **Serializer**：优先使用 annotate 或 prefetch 的预计算值，回退到查询

### 文件变更

| 文件 | 变更 |
|-----|------|
| `devices/views.py` | queryset 优化 |
| `devices/serializers.py` | 优先使用预计算值 |
| `sensors/views.py` | queryset 优化 |
| `sensors/serializers.py` | 优先使用预计算值 |

## 七、MQTT 主题自动生成修复

### 变更
