# 0.5 版本更新日志

## 变更概览

本次更新主要包含：**插件系统**（plugins 目录 + Plugin 登记表 + sync 命令 + 启用切换 API）、内置 `data_viz` 数据可视化插件、前端插件中心页与可视化视图、Claude 风格界面延续。

---

## 一、插件系统（核心新增）

### 背景

平台希望在不侵入核心 app 的前提下，挂载实验性 / 工具型功能（数据可视化、报表、推理等）。0.5 引入 `plugins/` 子目录约定 + 文件系统驱动的发现机制。

### 变更

- **新增** `iot_control_platform/plugins/` 子目录，作为所有插件的根
- **新增** `plugins/__init__.py`：提供 `discover_plugins()` 文件系统扫描和 `enabled_plugin_names()` 启用查询
  - DB 优先，DB 不可用时回退到 `plugin.json` 默认值
- **新增** 插件清单格式 `plugin.json`：`name / version / description / enabled` 四字段
- **修改** `config/settings.py`：自动把发现到的所有插件加入 `INSTALLED_APPS`，保证 makemigrations 可用
- **修改** `config/urls.py`：新增 `_mount_enabled_plugins()`，仅启用的插件挂载到 `/api/plugins/<name>/`，单个失败被 try/except 隔离

### 关键设计

- **零侵入**：扔进 `plugins/<name>/` + `plugin.json` 即可被识别，不改核心代码
- **失败隔离**：发现 / `INSTALLED_APPS` 注入 / URL 挂载三处都用 try/except 包裹
- **重启生效**：`enabled` 切换不热加载，Django URL conf 仅在启动时绑定
- **路径硬性约束**：所有插件 API 走 `/api/plugins/<name>/`，避免污染主路由

### 文件变更

| 文件                                                      | 变更                                       |
|-----------------------------------------------------------|--------------------------------------------|
| `plugins/__init__.py`                                     | 新增（discover / enabled_plugin_names）     |
| `plugins/README.md`                                       | 新增                                       |
| `config/settings.py`                                      | INSTALLED_APPS 注入                        |
| `config/urls.py`                                          | `_mount_enabled_plugins`                   |

## 二、Plugin 登记表与同步命令

### 背景

仅靠文件系统识别不够：需要在 DB 中持久化用户对每个插件的启用决策，且让前端可以在线管理。

### 变更

- **新增** `platform_settings.models.Plugin` 模型
  - 字段：`name`（unique）、`enabled`、`version`、`description`、`installed_at`、`updated_at`
  - 迁移：`platform_settings/migrations/0003_plugin.py`
- **新增** `python manage.py sync_plugins` 管理命令
  - 新增的插件按清单 `enabled` 默认值登记
  - 已存在的插件**仅刷新** `version / description`，**保留**用户的 `enabled` 决策
  - `--prune` 选项：移除文件系统中已不存在的插件登记
- **新增** Django Admin 注册 (`PluginAdmin`)：列表页可直接 `list_editable` 切换 `enabled`

### 文件变更

| 文件                                                                  | 变更        |
|-----------------------------------------------------------------------|-------------|
| `platform_settings/models.py`                                         | 新增 Plugin |
| `platform_settings/migrations/0003_plugin.py`                         | 新增        |
| `platform_settings/admin.py`                                          | 注册 PluginAdmin |
| `platform_settings/management/commands/sync_plugins.py`               | 新增        |

## 三、插件管理 API

### 变更

- **新增** `PluginViewSet`（`/api/plugin-manager/`）
  - `GET list / retrieve`：已认证用户可读
  - `POST sync/`：仅超级用户，触发 `sync_plugins` 命令
  - `POST <name>/enable/` / `<name>/disable/`：仅超级用户，切换启用状态（需重启进程生效）
- **新增** `PluginSerializer`：除 `enabled` 外全部 read-only，避免常规 PATCH 改 version/description
- **复用** `IsSuperuser` 权限类（沿用 platform_settings 模式）

### 文件变更

| 文件                                  | 变更                              |
|---------------------------------------|-----------------------------------|
| `platform_settings/views.py`          | 新增 `PluginViewSet`               |
| `platform_settings/serializers.py`    | 新增 `PluginSerializer`            |
| `platform_settings/urls.py`           | 注册 `plugin-manager` router       |

## 四、内置插件 data_viz

### 背景

作为插件系统的首个落地，提供"按时间窗查看 sensor / device 时序数据"的能力。

### 变更

- **新增** `plugins/data_viz/`（`apps.py` / `urls.py` / `views.py` / `plugin.json` v0.2.0）
- 三个端点（`/api/plugins/data_viz/<...>`）：

  | 端点         | 说明                                                          |
  |--------------|---------------------------------------------------------------|
  | `ping/`      | 端到端探针                                                    |
  | `sources/`   | 列出所有 sensor + device，附带可绘字段（来自 type 定义）       |
  | `series/`    | 取指定来源在时间窗内的时序数据 + 状态事件                      |

- 截断策略：超过 `limit`（默认 2000，上限 10000）时仅保留最近 `limit` 条，并标 `truncated=true`
- 仅只读核心模型（`Sensor / SensorData / SensorStatusCollection / Device / DeviceData`）

## 七、升级步骤

### Docker 部署

```bash
# 拉新代码后
docker compose down
docker compose up -d --build
# backend 启动会自动跑 migrate（应用 0003_plugin）
# 首次启动会通过 INSTALLED_APPS 自动发现 data_viz；
# 但要让前端"插件中心"显示数据，还需要触发一次 sync：
docker compose exec backend python manage.py sync_plugins
```

### 本地部署

```bash
git pull
cd iot_control_platform
python manage.py migrate              # 应用 0003_plugin
python manage.py sync_plugins         # 把 data_viz 登记到 DB
# 重启 runserver / gunicorn
```

### 前端

```bash
cd frontend
npm install                           # 增加了 echarts 依赖
npm run build                         # 或 npm run dev
```

## 八、设计要点速查

1. **插件 = 一个 Django app**，目录名即插件名，`apps.py:name = "plugins.<目录名>"`
2. **plugin.json 是真源**，sync 仅刷新 version/description，**保留**用户的 `enabled` 决策
3. **enabled 切换不热加载**，URL conf 在启动时绑定一次，必须重启进程
4. **所有发现到的插件**都在 `INSTALLED_APPS` 里，仅 URL 层按 `enabled` 过滤
5. **路径前缀固定** `/api/plugins/<name>/`，插件 `urls.py` 写相对路径

---

## 九、已知限制

- 不支持热插拔：`enabled` 改完必须重启 Django 进程
- 前端菜单为静态注册：新增插件需要在 `router/index.js` 与侧边栏手动加入口（后续可由清单驱动）
- 插件之间无依赖声明：加载顺序仅按目录字典序

详见 [plugins_design.md §九](../backend/backend_design/plugins_design.md)。
