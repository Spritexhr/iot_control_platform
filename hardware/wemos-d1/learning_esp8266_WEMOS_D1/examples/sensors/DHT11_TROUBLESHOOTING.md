# DHT11 传感器故障排除指南

## 🔧 常见问题：传感器读取失败

如果串口显示 `✗ DHT传感器读取失败！`，请按照以下步骤排查：

---

## 1. 硬件连接检查

### 标准连接方式

```
DHT11 传感器          WEMOS D1
─────────────────────────────────
VCC (红色)    →       3.3V 或 5V
GND (黑色)    →       GND
DATA (黄色)   →       D2 (GPIO4)
```

### ⚠️ 重要：上拉电阻

**DHT11 必须在 DATA 引脚和 VCC 之间连接一个 4.7KΩ - 10KΩ 的上拉电阻！**

```
VCC ──┬── 4.7KΩ ──┬── DATA (D2)
      │           │
      └───────────┘
```

**如果没有上拉电阻，传感器将无法正常工作！**

---

## 2. 引脚选择

### 推荐引脚（按优先级）

| WEMOS D1 引脚 | ESP8266 GPIO | 说明 | 推荐度 |
|--------------|--------------|------|--------|
| D2 | GPIO4 | 默认选择，I2C SDA | ⭐⭐⭐ |
| D5 | GPIO14 | SPI CLK | ⭐⭐⭐ |
| D6 | GPIO12 | SPI MISO | ⭐⭐ |
| D7 | GPIO13 | SPI MOSI | ⭐⭐ |
| D1 | GPIO5 | I2C SCL | ⭐ |

### ⚠️ 避免使用的引脚

- **D0 (GPIO16)**: 不能用于中断，不推荐
- **D3 (GPIO0)**: 启动模式选择，可能影响上传
- **D4 (GPIO2)**: 内置LED，可能干扰
- **D8 (GPIO15)**: 启动时下拉，不推荐
- **RX/TX**: 串口通信，避免使用

### 更换引脚方法

在代码中修改：

```cpp
// 从 D2 改为 D5
#define DHTPIN D5  // 或使用 GPIO 编号：14
```

---

## 3. 传感器类型确认

### DHT11 vs DHT22

确保代码中的传感器类型与实际硬件匹配：

```cpp
#define DHTTYPE DHT11   // 如果是 DHT11
// 或
#define DHTTYPE DHT22   // 如果是 DHT22
```

**区别：**
- DHT11: 温度范围 0-50°C，精度 ±2°C
- DHT22: 温度范围 -40-80°C，精度 ±0.5°C

---

## 4. 供电检查

### 电压要求

- **DHT11**: 3.3V - 5.5V
- **WEMOS D1**: 3.3V 逻辑电平

### 供电方式

1. **USB 供电**（推荐）
   - 通过 Micro USB 连接电脑或充电器
   - 确保电源稳定（至少 500mA）

2. **外部供电**
   - 使用 3.3V 稳压电源
   - ⚠️ **不要使用 5V 直接供电给 GPIO！**

### 检查方法

用万用表测量：
- VCC 和 GND 之间应该有 3.3V 或 5V
- DATA 引脚空闲时应该接近 VCC（因为有上拉电阻）

## 6. 常见错误和解决方案

### 错误 1: 一直显示读取失败

**可能原因：**
- ❌ 缺少上拉电阻（最常见！）
- ❌ 引脚连接错误
- ❌ 传感器损坏

**解决方案：**
1. 添加 4.7KΩ 上拉电阻
2. 检查所有连接
3. 尝试更换传感器

### 错误 2: 偶尔读取失败

**可能原因：**
- ❌ 供电不稳定
- ❌ 读取间隔太短（DHT11 需要至少 2 秒间隔）

**解决方案：**
1. 使用稳定的 USB 电源
2. 增加 `delay()` 时间到 2000ms 或更长

### 错误 3: 读取值异常（如 0 或 255）

**可能原因：**
- ❌ 传感器类型设置错误（DHT11 vs DHT22）
- ❌ 引脚冲突

**解决方案：**
1. 确认传感器类型
2. 更换引脚

---

## 7. 硬件测试步骤

### 步骤 1: 检查连接

1. 断开所有连接
2. 用万用表检查传感器是否正常
3. 重新连接，确保牢固

### 步骤 2: 测试上拉电阻

1. 测量 DATA 引脚电压
2. 空闲时应接近 VCC（3.3V 或 5V）
3. 如果接近 0V，说明上拉电阻未连接或连接错误

### 步骤 3: 更换引脚测试

1. 尝试使用 D5 引脚
2. 修改代码：`#define DHTPIN D5`
3. 重新上传测试

### 步骤 4: 更换传感器

如果以上都正常，可能是传感器损坏，尝试更换新的传感器。

## 8. 完整连接图

```
                    WEMOS D1
                    ┌─────────┐
                    │         │
     DHT11          │   D2    │←─── DATA (黄色线)
    ┌─────┐         │         │
    │ VCC │─────────┤ 3.3V    │
    │     │         │         │
    │ GND │─────────┤ GND     │
    │     │         │         │
    │ DATA│─────────┤ D2      │
    └──┬──┘         └─────────┘
       │
       │ 4.7KΩ 上拉电阻
       │
       └─────────── 3.3V
```

## 9. 快速检查清单

- [ ] 上拉电阻已连接（4.7KΩ - 10KΩ）
- [ ] VCC 连接到 3.3V 或 5V
- [ ] GND 连接到 GND
- [ ] DATA 连接到 D2（或其他可用引脚）
- [ ] 传感器类型设置正确（DHT11/DHT22）
- [ ] 供电稳定（至少 500mA）
- [ ] 引脚选择正确（避免 D0, D3, D4, D8）
- [ ] 代码中引脚定义正确
- [ ] 串口波特率设置为 115200

---

## 10. 如果仍然无法解决

1. **检查库安装**
   - 确保安装了 DHT sensor library
   - 工具 → 管理库 → 搜索 "DHT sensor library"

2. **尝试其他引脚**
   - D5 (GPIO14) - 最稳定的替代选择

3. **检查传感器**
   - 用万用表测试传感器是否损坏
   - 尝试更换新的传感器

4. **查看详细错误**
   - 使用改进的调试代码查看更详细的错误信息
